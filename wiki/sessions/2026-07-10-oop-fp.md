# Session: OOP & Functional Programming (2026-07-10)

Closed **P1 #4** on [[target-role-profile]] — the JD's "OOP + functional programming principles." New
`fe-paradigms` module, 5 lessons, built to mirror the `fe-typescript` shape (2 live+tested, 3 concept).
Tests **138 → 148**, build clean, `node --check` clean.

## What shipped — `fe-paradigms` (frontend, level 3)
Placed in `curriculum.js` after `fe-js-core`, before `fe-react` (paradigms build on JS core, precede React).

1. **`fe-para-intro`** (concept) — JS is multi-paradigm. Imperative vs declarative; OOP = state + behavior
   bundled; FP = pure functions over immutable data. Ties to React ("UI is a pure function of state").
2. **`fe-fp-pure`** (utility/vanilla, **live + tested**) — pure = deterministic + no side effects. Immutable
   cart ops: `addItem` (spread), `setQty` (map + object spread, unchanged items reused by reference),
   `removeItem` (filter), `total` (reduce). Tests assert no mutation + new references + correct values.
3. **`fe-fp-compose`** (utility/vanilla, **live + tested**) — first-class fns, HOF, map/filter/reduce,
   `pipe` (L→R via reduce) and `compose` (R→L via reduceRight). Tests assert ordering + `slugify` +
   `sumOfEvenSquares` = 56.
4. **`fe-oop-classes`** (concept) — classes bundle state+behavior; `#private` fields (ES2022) enforce
   encapsulation; methods live on the shared prototype; **composition over inheritance** (GoF; React docs).
5. **`fe-solid`** (concept) — SOLID taught as front-end *instincts* (split responsibilities into hooks/pure
   fns; extend via props not `if`; inject dependencies for testability), not acronym recitation.

## Role framing applied (per [[target-role-profile]])
- **React angle** — immutability → new references → re-render; render purity.
- **AI-audit angle** — flagged the two most common generated-code smells this module counters: mutating a
  caller's input, and spinning up needless class hierarchies.
- **Performance note** — each `map`/`filter` is a new array + a pass; fuse to one `reduce`/loop only on hot
  paths when profiling says so.
